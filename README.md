# Personal Site (Instagram-style)

Статичный личный сайт-профиль в стиле Instagram: шапка профиля, лента историй,
лента публикаций 3×3, модальное окно просмотра поста, а также дополнительные
вкладки — «Цели», «My Life» (колесо баланса) и «Travel» (карта посещённых стран).

## Запуск

Открыть `index.html` в браузере (двойной клик), либо поднять локальный сервер:

```bash
python3 -m http.server 8000
```

и перейти на `http://localhost:8000`.

## Структура

```
├── index.html                    # разметка страницы
├── assets/
│   ├── css/style.css             # все стили
│   ├── js/
│   │   ├── script.js             # данные и логика (табы, модалка, рендер)
│   │   └── world-map-data.js     # SVG-контуры стран для вкладки Travel
│   └── images/
│       ├── posts/                # сюда положить свои фото постов
│       ├── stories/               # сюда положить аватарки для историй
│       └── logos/                 # сюда положить логотипы брендов (раздел «Бизнес»)
```

## Как наполнить своим контентом

Всё редактируется в начале `assets/js/script.js`:

1. **Профиль**: имя пользователя, био и ссылка — прямо в `index.html` (секция `.profile-header`).
2. **Публикации**: массив `POSTS`. Если положить фото в `assets/images/posts/` и указать путь в поле `image`, вместо цветной заглушки покажется реальное фото.
3. **Истории**: массив `STORIES` — подписи и эмодзи; пункту можно задать `tab`, чтобы клик по нему переключал на нужную вкладку профиля.
4. **Цели**: массив `GOALS` — прогресс от 0 до 100 и срок.
5. **My Life**: массив `LIFE_AREAS` — 8 сфер жизни с оценкой 1–10 (колесо баланса).
6. **Travel**: `TRAVEL_DATA` в `script.js` — соответствие кода страны (ISO 3166-1 alpha‑2, например `ru`, `jp`) статусу из `TRAVEL_STATUSES`.
7. **Бизнес**: массив `BUSINESS_PROJECTS`. По умолчанию логотип рисуется текстом (`logoClass`, стили в `.brand-logo--*` в `style.css`) — так на сайте не используется чужая товарная графика. Чтобы поставить настоящий логотип бренда, положи файл в `assets/images/logos/` и укажи путь в поле `logoImage` (он приоритетнее `logoClass`).

Сайт полностью статический — HTML/CSS/JS без сборки, можно разместить на GitHub Pages, Netlify или любом хостинге.
